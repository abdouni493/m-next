## 📊 DATABASE SCHEMA

### **1. USERS TABLE** (Extended Authentication)
Extends Supabase Auth with custom user profile data
- User ID (UUID, linked to auth.users)
- Email (unique)
- Username (unique)
- Role (admin/employee)
- Phone, Address, Salary
- Hire Date, Avatar URL
- Is Active flag
- Timestamps

```sql
-- Access user data
SELECT * FROM users WHERE id = auth.uid();
```

### **2. CATEGORIES TABLE**
Product categories
- ID (UUID)

- Description
- Timestamps

## ⚙️ TRIGGERS & FUNCTIONS

### **Trigger: update_product_quantity()**
Automatically updates product current_quantity when invoice items are added:
- Sale = -quantity
- Purchase/Stock = +quantity

### **Trigger: create_audit_log()**
Automatically logs all INSERT/UPDATE/DELETE operations on:
- products
- invoices
- suppliers

Stores old and new values in JSONB format for compliance.

## 🔐 ROW LEVEL SECURITY (RLS)

All tables have RLS enabled with policies:

### **READ Policies**
- Users can read all data (public access for most tables)
- Users can only read their own data (users table)

### **MODIFY Policies**
- Only admins can modify tables (via role check)
- Checked on INSERT/UPDATE/DELETE

### **Admins can:**
- Create/Edit/Delete products
- Manage suppliers
- Create/Manage invoices
- Manage employees
- Generate reports
- Manage barcodes

### **Employees can:**
- View products, suppliers, customers
- View sales invoices
- View their own data

## 🔑 AUTHENTICATION FLOW

### **Signup Process**
1. User enters username, email, password
2. System validates inputs
3. `signUp()` creates auth user in Supabase Auth
4. Creates user profile in `users` table with role = 'admin'
5. Returns user object and auth user
6. User automatically logged in
7. Redirected to Dashboard

### **Login Process**
1. User enters email and password
2. `signIn()` authenticates with Supabase Auth
3. Retrieves user profile from `users` table
4. Returns user object
5. User logged in
6. Redirected to Dashboard

### **Session Management**
- Supabase handles session tokens
- LocalStorage stores user profile
- Auth state persists across page refreshes
